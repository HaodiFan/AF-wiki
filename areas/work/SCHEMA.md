# Work Area Schema

> Purpose: provide a stable home for ongoing work context, work-facing operating assumptions, and future work area expansion.
> Owner: AF
> Last updated: 2026-04-20

## Why this is an area

Work is an ongoing responsibility, not a single project.
This area exists so future work notes do not get mixed into:
- personal fitness tracking
- retained technical reading notes
- shared capture pipelines

## Preferred skill routing

- `second-brain-wiki-architecture`:
  - use for early-stage structure definition and work area reorganization
- work-specific planning or software-development skills:
  - add them here once the work area's recurring workflows become clearer

## Current structure

```text
work/
├── SCHEMA.md
├── index.md
├── 00-active-context.md
└── 99-change-log.md
```

## Expansion rule

As work content grows, add only the folders that are justified by real usage, for example:
- project trackers
- standing operating notes
- recurring meeting or decision logs

Do not overbuild this area before it has real content.
